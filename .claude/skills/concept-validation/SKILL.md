# Concept Validation

## Check 1: Problem Clarity
Can you describe the problem in one sentence a 12-year-old would understand?
If not: the problem isn't clear enough to build for.

Common failure: "People want to be more productive" is a desired state, not a problem.
Fix: Interview 3 real potential users. Use their words.

## Check 2: Market Reality
- Who are the top 3 competitors? (If you can't name them, research more.)
- What do their one-star reviews say? (This is the gap map.)
- Why hasn't any competitor fixed this gap?

**Red ocean signal:** Top 3 have >1M downloads with identical features.
**Blue ocean signal:** Consistent unaddressed complaint pattern across competitors.

## Check 3: User Specificity
"Young professionals" = 50M people = too broad.
"25-32 year old first-time managers in their first 6 months" = 500K people = right size.

## Check 4: Retention Mechanic
What happens if the user opens the app today but not tomorrow?
If nothing changes: the app has no pull.

Strong mechanics: something accumulates, something is time-sensitive, social accountability, professional consequence.

## Check 5: Monetization Reality
Consumer apps: $3-10/month subscription sweet spot.
Prosumer: $10-30/month. B2B: $20-100+/month per seat.

Paywall position: after value is demonstrated, not before.

## Check 6: Scope Realism
List all v1 features. Estimate build time. Sum.
If sum > 12 weeks solo: scope is too large.

Cut framework: remove anything the user can work around, anything requiring another person, anything only 20% of users need.

## Minimum Lovable Product
Smallest version a real user would actually pay for. Not minimum viable (engineer's test). Minimum lovable (user's test). Usually: core loop + one feature that makes the user feel understood.
